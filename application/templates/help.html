Frequently Asked Questions (FAQ)

General Questions

The SCFAMI Dashboard is a Web platform for grid operators to collect data files from the Smart Meter PC. It offers features such as Data Transfer, Application Launch, Spider Integration and Admin Configurations.

The ultimate goal of this platform is to provide grid operators with the visibility to detect and analyse attacks on the Smart Meter Network System.

This platform offers 4 key features for grid operators:

Data Transfer: Operators can choose to transfer data files at the point of request, or schedule them for recurrent transfers.
Application Launch: Operators can launch Desktop applications or Arkime here.
Spider Integration: Operators can schedule Spiders (Web crawlers) to crawl URLs for YARA rules.
Admin Configurations: Operators can configure certain administrative settings of the SCFAMI Dashboard.

Any mentions of data files on this Help page refer to the following: Smart Meter data files, Windows Event (System and Security) logs, KEPServer logs and Packet Capture (PCAP) files stored on the Smart Meter PC.


Admin Configurations Questions

Grid operators can update the IP address of the Smart Meter PC, the credentials to use for the Smart Meter connection, and the maximum number of multi-threaded workers the system can use.

Grid operators can update the directory to open via Nautilus File Manager.

In the Arkime Views page, Arkime's API is accessed to dynamically populate the page with Arkime views. This API requires login credentials that are set on Arkime itself. Thus, grid operators have the ability to update this on the SCFAMI Dashboard as well.


Data Transfer Feature

This page offers grid operators a feature to transfer data files from the Smart Meter PC at the point of request, or schedule them for recurrent transfers.

Upon selecting a type of data file, a Web form will be displayed for operators to fill up. The form consists of the following fields: Transfer Type, Date, Start Time, End Time, and Timezone.

When the Transfer Type is selected as Now, the specified range of data files will be transferred now.

Conversely, when the Transfer Type is selected as Schedule, the specified range of data files will be transferred at the recurring schedules.
For this option, the baseline for Transfer Frequency is Daily. It requires the start time of the data files, the duration to download for, and a job name to identify this job. The recurring download folder will be renamed to the job name specified here. Weekly, on top of the Daily parameters, requires operators to select the days of a week to run this job schedule, while Monthly requires operators to select the day of each month.

The download path of the data files will be shown after the download process has completed.

A side feature of this Ubuntu machine is that there are scheduled jobs that run periodically to remotely capture network traffic on the Honeypot machine.

This page serves to open the directory of the captured files on the Nautilus File Manager.

Data Transfer jobs that are scheduled will be displayed as a table on this page. Grid operators can choose to delete any scheduled job, disable a current job, or enable a disabled job.

Note that all jobs are automatically set to enabled when they are created.


Application Launch Questions

5 open-source Desktop GUI applications are identified for grid operators to use. They are FileZilla, Wireshark, Zui, Dbeaver and Remmina. They can be launched from the click of their respective buttons.

Filezilla: Enables operators to manage data files using a comprehensive interface.
Wireshark: Allows operators to analyse PCAP files or network traffic.
Zui: A Desktop application for Brim which is used to explore and analyse structured data files.
Dbeaver: A SQL client software that is meant to query jobs created by the Spider Integration explained below.
Remmina: A RDP client that provides operators with an option to connect remotely to the Smart Meter PC.

Arkime allows operators to see exactly what is going on from a network point of view.

Grid operators can click on any of the buttons to open a new tab with Arkime Sessions with the filtered view. These views are created on Arkime itself.


Spider Integration Questions

Spiders, in this context, are Web crawlers designed to browse user-specified URLs for YARA files.

Grid operators can specify a URL to crawl, the depth of the crawling and the Spider to use.

Spider jobs that are Running or Completed will be displayed as their respective tables on this page. Grid operators can view these jobs from the two tables.

For the Completed table, a default filter is provided to showcase the details of jobs with unique URLs. The filter is dynamic and operators can filter by specific URLs that were scheduled.
Operators can also query for these jobs via Dbeaver mentioned in Local Applications above.
